## User behaviour notes

User behaviour should be influenced by the design of a system, rather than rules put atop it. Users will always give into their basest instincts, regardless of rules put into place. These guidelines are built upon that principle.

These guidelines results in a design that reduces mental load as much as possible. The initial poster must decide which channel to post in, and if it is too difficult to find an initial channel, then they will err towards not posting. Initial posters are exceedingly important to spark conversation within your server. Simply freely allowing them to choose the channel which seems most accurate is sufficient, it should not be a great ordeal.

Minimizing the number of channels means that both moderators and users do not need to flick so many channels to find something interesting. It means that there are more active and recent conversations compared to "Dead" channels.

Correctly seperating channels means that users can filter out content they are uninterested in, and so can get more out of your server. Correctly seperating categories means that users can hide/show entire categories they are uninterested in.

## Admin Activity Guidelines

1. Begin from #general, and observe conversations. 
    - If a topic comes up a significant portion of the time, with a small number of participating users (niche appeal), consider creating a new channel for it.
    - Do not jump the gun: channels should reflect real activity, not desired activity. Over-organisation is death.
    - If there is a "dead" channel, consider replacing it if you need to create a new channel rather than adding a new one whole-sale.
    - New channels merely segregate an initial post about a topic from #general. If something is of actual general appeal, it should stay in #general.
2. Be permissive with people using channels; build up psychological safety. 
    - People shouldn't feel like they are doing something wrong. Allow conversations to veer off-topic, to a certain extent. 
    - Moving a conversation is effectively killing it; saying it is "unwelcome".
3. If someone asks for a new channel...
    - They are just asking if what they want to talk about is welcome. Simply ask them to use an existing channel, or broaden an existing channel's scope to include their request.
    - Do not create new channels on request. Use your best judgement if it is actually required.
4. Keep channels For Humans
    - Bots should not have a significant presence in the general channels, neither should general admin messages pollute them.
    - These messages almost never spark conversation, and make it difficult to start them.

## Channel Requirement Rules

1. Servers should have these categories as a given, which can be ignored for daily use:
    1. an admin category. May split into a (read-only & announcements) and (actions) categories. Include automated welcome messages, bot channels in here.
    2. a hidden management category
2. Other categories can be called "content" categories, and are the bread-and-butter.
3. Categories should be small. A "content" category must have 3-5 channels.
4. Categories should be congruent. Each category should have a theme tying its channels together.
5. Channels should be obvious. 
    1. Its content and purpose should be clear from the title, which should have 1-2 terms total. 
    2. Additional disambiguation should be done in the description and not pinned messages.
6. Channels should be active. Activity can be defined as two or more users posting or reacting in it at least every week. "Dead" channels should be re-integrated.
7. Channels should be "chat" or "post" channels.
8. Channels should be relevant. Each channel must fit the overall vibe and purpose of the server and its users.
9. Channels should NEVER use the forums feature, because it will kill activity. The only time this is acceptable is if the whole server runs on the forums feature.
